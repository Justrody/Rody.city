Boas praticas de UI/UX

post: 29/08/2025

inicio 26/08/2025
Existem 10 eurísticas que você pode analisar em uma interface.

1. Visibilidade e status no sistema (sinalização sobre onde você está no sistema)
2. Correspondência (ícones relacionáveis ao mundo real ex: lixeira para deletar, lupa para pesquisar, etc)
3. Controle e liberdade do usuário (união dos conceitos anteriores e execução para que ele possa desfazer caso mude de ideia)
4. Consistência e padronização (manter tudo padronizado para a consistência)
5. Prevenção de erros (prevenir erros do usuário)
6. Reconhecimento e memória (deixar de maneira reconhecível para maior conforto)
7. Estética e design (evitar informações desnecessárias, menos é mais)
8. Recuperação de erros (deixar visível o erro do usuário e trazer correção para o próprio)
9. Eficiência e flexibilidade (regras únicas que funcionam de mesmo modo)

Menos é mais.
Minimalismo é de grande necessidade para um bom design, com o minimalismo é possível unir forma à função de maneira que, quando satisfatória, entrega não só uma boa experiência de usuário mas também um bom layout, que na psicologia do design é humanamente agradável e esteticamente bonito (harmônico).

Consistência.
Um design consistente é um design padronizado. A padronização do design para interfaces é de suma necessidade para que ela seja agradável e intuitiva.
Espacamentos diferentes para subcategorias, cores padronizadas de fundo, etc, são de grande importância para a visualização humana.
Consistência na acessibilidade também é de grande importância, pois os humanos não só utilizam o visual mas também o auditivo e o layout intuitivo. Manter-se consistente é ser responsável com todos, sejam pessoas portadoras de deficiência ou não.
A consistência cria regras e padrões: espaçamento entre elementos iguais dentro de formulários, separação de categorias, nomes utilitários, heurísticas organizadas, cores de fundo similares, sites diferentes com campanhas diferentes.

Respeito de espaços.
O respeito a espaçamentos é um conceito visual que abrange todos os citados acima. Espaçamentos são necessários no texto para definir palavras diferentes, argumentos e introduções em parágrafos, evidenciando que espaços (void) são necessários para a visão humana e interpretação da diferença/separação de objetos, informações, números, etc.

O impacto das fontes.
As fontes são de suma importância como tudo o citado acima para sua interface. No design, a fonte ao ser reduzida para determinados tipos de leitura é essencial para manter a legibilidade e o impacto, sendo marcante e eficiente.

Espaço não é dinheiro.
O espaçamento é decisivo para uma UI agradável. Para que se mantenha de forma adequada a seus espaços, a UI deve respeitar algum padrão espacial com vazios para evitar cansaço visual do usuário, tornando a UI limpa e a UX agradável e eficiente.

Ritmo no design.
Estruturas que se repetem e que são comuns de maneira organizada para que seja rítmica e eficiente. A ideia é reduzir a carga cognitiva do site por meio de padrões lógicos (ou não) que familiarizam o usuário nela.
Ou seja, consistência é essencial para um design eficiente e uma boa experiência de usuário.

Escalonamento de textos.
Sequências numéricas de dobro ou valores fixos mínimos.
O escalonamento de texto é essencial para uma boa responsividade e para uma visualização confortável do site: de 8 para 16, 16 para 24, 32, 40, etc, assim mantendo o site eficiente. Busque o conforto e a eficiência unidos em uma coisa apenas, mantendo o mesmo padrão para manter o peso adequado ao usuário.

Grid no Figma.
Grid no Figma é essencial para a padronização. Ao utilizar o Figma, você seleciona em preferências e coloca o nugget amount; lá você seleciona de small até big. Ou seja, ao utilizar isso ele vai jogar de acordo com o número de pixels para a escala.
